# Протокол синхронизации (SYNC_PROTOCOL)

Данный протокол описывает ежедневную рутину и порядок действий при обновлении ядра базы данных Katalog-AI. Если вы просите агента "Обновить индекс", он обязан действовать строго по этому регламенту.

## Шаг 1: Сверка (Inventory Check)

Перед началом любых коммитов необходимо проверить, что список всех `.md` файлов в корневой директории полностью соответствует массиву индексов в `catalog.json`. Никаких лишних файлов или недостающих записей быть не должно.

## Шаг 2: Синхронизация (Ecosystem Sync)

Обновить списки и ссылки в следующих файлах:

* `README.md`
* `index.html`
* `llms.txt`

**Требование к ссылкам:** Все ссылки должны быть СТРОГО относительными в формате `[Название](имя_файла.md)` (для Markdown) или `<a href="имя_файла.md">Название</a>` (для HTML).

## Шаг 3: Валидация (Sanity Check)

Убедиться в целостности графа: пробежаться по всем генерируемым путям и удостовериться, что нет битых ссылок (Path Validation). Если обнаружено несоответствие с правилами из `CONTRIBUTING.md` — агент обязан сообщить об ошибке **до того**, как будет выполнен пуш.

## Шаг 4: Отправка изменений (Commit & Push)

Только после прохождения шагов 1-3 разрешается фиксировать состояние.
Команды для фиксации:

```bash
git add .
git commit -m "[SYNC] Update catalog index"
git push
```
