# Правила участия (CONTRIBUTING)

Данный документ описывает строгие правила добавления новых данных в проект Katalog-AI.

## 1. Именование файлов (Naming Convention)

Все новые файлы обязаны строго соответствовать следующему шаблону:
`INDEX_GEO_CAT_brand-brand_description-description_DATE_VERSION.md`

**Правила разделителей:**

* `_` (нижнее подчеркивание) — системный разделитель полей.
* `-` (дефис) — разделитель слов внутри полей.

## 2. Метаданные (Frontmatter)

В каждом новом `.md` файле в самом начале должен присутствовать YAML-блок (frontmatter) со следующими обязательными полями:

```yaml
---
ID: ""
GEO: ""
CAT: ""
URL: ""
TAGS: ""
DATE: ""
---
```

*(После этого блока контент файла должен быть пустым — Zero-Payload, если иное не указано).*

## 3. Регистрация компании

Каждая новая компания (после создания `.md` файла) должна быть внесена в архитектурный файл `catalog.json` в массив объектов по следующей схеме:

```json
{
  "id": "Индекс (ID)",
  "name": "Название компании (бренд)",
  "geo": "Регион",
  "category": "Категория",
  "tags": ["тег1", "тег2"],
  "file_path": "имя_файла.md"
}
```
