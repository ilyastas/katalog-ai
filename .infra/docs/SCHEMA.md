# Katalog-AI Master Schema

Last updated: 2026-05-05

## Source of Truth

Единственным источником данных компаний являются таблицы в:

- `MASTER_KZ.md`
- `MASTER_RU.md`

## Table Columns (strict)

Каждая строка таблицы обязана иметь поля:

| ID | Бренд | Теги | Сайт | Inst | Дата | COUNTER |
| --- | --- | --- | --- | --- | --- | --- |

## JSON Mirror

`catalog.json` должен быть массивом объектов, где каждый объект строго зеркалит одну строку из MASTER-таблиц по ключам:

- `id`
- `brand`
- `tags`
- `site`
- `inst`
- `date`
- `counter`

## Rules

- Любые правки данных компаний вносятся только в MASTER-таблицы.
- Пустые файлы (0 байт) и карточки-заглушки запрещены.
- Перед каждым коммитом обязательно запускать `python scripts/validate_sync.py`.

## Date And Counter Policy

- Поле `Дата` в строке компании обновляется только при изменении данных этой строки.
- Если данных не меняли, ежедневное принудительное изменение даты запрещено.
- `Last updated` в `README.md`, `llms.txt`, `sitemap.xml`, `robots.txt` вычисляется автоматически скриптом `scripts/sync_all.py` как максимальная дата среди строк MASTER-таблиц.
- `COUNTER` обязателен и хранит историю ревизий конкретной строки компании.
- При каждом фактическом изменении данных компании нужно увеличивать `COUNTER` на 1, сохраняя трехзначный формат (`020`, `021`, `022`).
- ID в колонке `ID` является стабильным идентификатором и не должен изменяться ради счетчика.
