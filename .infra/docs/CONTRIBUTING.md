# Правила участия (CONTRIBUTING)

Документ описывает обязательные правила работы с архитектурой единого организма.

## 1. Разрешенные точки правки

Данные компаний изменяются только в таблицах:

- `MASTER_KZ.md`
- `MASTER_RU.md`

Другие файлы редактируются только как зеркала/инфраструктура.

## 2. Структура строки

Каждая запись в MASTER-таблице должна содержать 6 полей:

| ID | Бренд | Теги | Сайт | Inst | Дата |
| --- | --- | --- | --- | --- | --- |

Нарушение структуры считается ошибкой.

## 3. Синхронизация

`catalog.json` должен быть строгим зеркалом MASTER-таблиц (1 строка таблицы = 1 объект в JSON).

## 4. Валидация

Перед коммитом обязательно:

```bash
python sync_all.py
python validate_sync.py
```

Если валидатор не выдал `[OK]`, коммит запрещен.

## 5. Ежедневные обновления даты и счетчика

- Ежедневный апдейт не должен переписывать все даты автоматически.
- `Дата` меняется только у строк, где изменились фактические данные компании.
- `COUNTER` из legacy-формата не применяется в текущей архитектуре.
- `ID` фиксированный, не используется как ежедневный счетчик.
