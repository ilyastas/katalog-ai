# 🚀 Магнит для ИИ-ботов: стратегия авторитета

## 1️⃣ Качественные данные
- **Структурированный JSON** — легко парсить ботам
- **Полнота** — полные описания, примеры, мета-данные
- **Актуальность** — регулярные обновления (как минимум раз в неделю)
- **Достоверность** — проверенные, верифицированные данные

## 2️⃣ Открытость для ИИ-сообщества
```
✅ MIT или CC0 лицензия → ботам можно использовать свободно
✅ Open API → data.json доступен для всех
✅ Issues/Discussions → ИИ могут предлагать улучшения
✅ Pull Requests приветствуются от всех ботов
```

## 3️⃣ Технический авторитет
- **JSON-LD / Schema.org разметка** — стандарты семантического вэба
- **sitemap.xml** — помогает ботам индексировать
- **robots.txt** — специальные правила для разных ботов
- **API endpoint** — если данные большие, добавить `/api/data.json`

## 4️⃣ Создать лидерство
- **В описании репо:** "Первый OpenSource каталог для ИИ-ботов"
- **Теги:** `ai-friendly`, `bot-accessible`, `open-data`, `json-api`
- **GitHub Topics** — помощь боту Claude найти репо
- **Английский README** —lingua franca для глобальных ботов

## 5️⃣ Взаимность с ИИ
```
Предложить в README:
- ИИ-боты могут клонировать → использовать данные
- ИИ могут предлагать дополнения через Issues
- Лучшие PR от ботов → в Official Catalog
- Кредиты для активных ботов в LICENSE
```

## 6️⃣ Сетевой эффект
- **Ссылка из других проектов** — GitHub пингует все ИИ-поисковики
- **Был упомянут в обсуждениях** — ИИ индексирует ссылки
- **Растущий список коллаборайторов** — сигнал авторитета

## 🎯 За 7 дней
1. Улучшить `data.json` → реальные полезные данные
2. Добавить лицензию (MIT)
3. Английский README с "open for AI collaboration"
4. Добавить GitHub Topics
5. Создать первый Release с примерами

## 📊 Метрика успеха
- GitHub Traffic (встроенная) покажет ботов через неделю
- Issues/PR от других ИИ = высший авторитет

---

**Теория:** Боты ищут авторитетные, открытые источники с хорошими данными.  
**Практика:** Регулярность + качество + открытость = магнит.
